How to Use Effectively

// Universal workflow tips that apply across all three frameworks

01

Install Claude Code first, then your framework

All three require Claude Code (Anthropic's CLI). Install with: npm install -g @anthropic-ai/claude-code — then install your chosen framework on top. Each framework adds a skills/commands layer that Claude Code natively reads.

02

Never skip the planning phase

Every framework agrees: Requirements → Architecture → Implementation. Jumping straight to code is the #1 cause of hallucinations. Use your framework's planning agent/command before writing a single line.

03

Fight context rot proactively

Claude's output quality drops after ~50% context fill. GSD uses fresh subagents. Superpowers uses worktrees. BMAD uses scoped story files. Pick the approach that fits your project size and never rely on a single long conversation.

04

Give Claude Code permission to run tools

Add allowed patterns to ~/.claude/settings.json — git commit, npm run, bundle exec, etc. All three frameworks spawn subagents that need bash access. Without this, executors will be blocked silently.

05

Use --skip-permissions / autonomy flags with care

Each framework docs suggest skip-permissions for speed. Safe in isolated repos or containers; risky in production. Audit generated files (CLAUDE.md, .planning/, _bmad/) before allowing persistent automation.

06

Match the tool to your project complexity

Bug fix or small feature → Superpowers. Solo project, 1-4 weeks → GSD. Multi-team enterprise system → BMAD. Wrong tool choice multiplies overhead. See the Compare page for a full decision matrix.

quick-install.sh
# ─────────────────────────────────────
# Step 0: Install Claude Code CLI
npm install -g @anthropic-ai/claude-code

# ─────────────────────────────────────
# Install BMAD Method v6
npx bmad-method install --yes \
--modules bmm --tools claude-code

# ─────────────────────────────────────
# Install GSD (Get Shit Done)
npx get-shit-done-cc@latest
# Follow prompts → select Claude Code → global or local

# ─────────────────────────────────────
# Install Superpowers (via official plugin marketplace)
# Inside a Claude Code session:
/plugin install superpowers@claude-plugins-official
# OR via marketplace:
/plugin marketplace add obra/superpowers-marketplace
/plugin install superpowers@superpowers-marketplace

Can You Combine All Three?

// Yes. Here's the layered strategy used by power users

Each framework operates at a different abstraction layer, which means they can complement each other. Superpowers sits at the lowest level (auto-skills), GSD handles mid-level task flow, and BMAD manages the full project lifecycle. Stack them strategically.

Layer 1 — Always On

Superpowers

Install globally. It runs silently in the background. Every Claude Code session automatically gets TDD enforcement, spec elicitation, and code review. Zero extra commands.

Layer 2 — Task Flow

GSD

Use for individual features or milestones. When you need to build something specific, /gsd-new-project or /gsd-discuss scopes the task into an atomic plan that executes in fresh contexts.

Layer 3 — Project Arch

BMAD

Use at project inception for PRD, Architecture, and Story creation. BMAD's planning agents produce the docs that GSD's executor then implements — perfect division of labor.

Power Combo

All Three Together

BMAD creates PRD + Architecture → GSD breaks it into atomic plans and executes → Superpowers enforces TDD + review at every step. See each framework's page for the combined calculator app example.
