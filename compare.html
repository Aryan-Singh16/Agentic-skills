Who They Are
🔥 BMAD Method
Breakthrough Method for Agile AI-Driven Development
by BMad Code Organization
42.1k ★ MIT License Node 20+
BMAD simulates a full agile development team with 21 specialized AI agents. It mirrors how real software teams work — from business analyst to architect to developer to QA — with structured handoffs between agents. Perfect for projects that need real documentation, full planning cycles, and enterprise-level rigor.
⚡ GSD
Get Shit Done — Meta-prompting, Context Engineering & Spec-Driven Dev
by TÂCHES (glittercowboy)
58k ★ MIT License 15 runtimes
GSD is a philosophy as much as a framework: keep it light, eliminate context rot, ship small verified increments. It uses six surgical commands to take you from raw idea to shipped code, spawning fresh 200k-token subagents for each task so quality never degrades. Built for creative solo devs who don't want sprint ceremonies.
🦸 Superpowers
Agentic Skills Framework & Software Dev Methodology
by Jesse Vincent (obra) · Prime Radiant
176k ★ MIT License Official Plugin
Superpowers takes a radically different approach: skills trigger automatically. You don't invoke it — it just works. The moment you describe something you want to build, Superpowers pulls out a spec, gets your approval, writes a clear plan, then dispatches subagents. Strong emphasis on TDD, code review, and YAGNI principles.

When to Pick Each

🔥 Choose BMAD when...

  • Building an enterprise-scale app from scratch
  • You need a full PRD + architecture + sprint plan
  • You want multiple specialized agents collaborating
  • Your team needs structured handoffs and documentation
  • Working on non-code domains (writing, business plans)
  • You want scale-adaptive planning (Level 0-5)

⚡ Choose GSD when...

  • You're a solo developer shipping fast
  • Context rot is breaking your current workflow
  • You want atomic, verifiable commits per feature
  • You hate sprint ceremonies and enterprise theater
  • Building something you need shipped this week
  • You want GitHub/Linear issue-driven development

🦸 Choose Superpowers when...

  • You want TDD without thinking about it
  • You keep forgetting to write tests or review code
  • You want a "set and forget" quality enforcer
  • Working across multiple AI IDEs (Cursor, Codex, etc.)
  • You want to start immediately with no learning curve
  • You need git worktrees for isolated feature dev

Ultimate Stack — All Three Together
Yes — They Stack Beautifully

Each operates at a different layer. There is no conflict.

LAYER 1 (Always Active)
Superpowers — TDD, spec elicitation, code review, worktrees
→ No action needed. Just installed globally.

LAYER 2 (Project Planning)
BMAD — /analyst /pm /architect → PRD, Architecture, Stories
→ Run once at project start. Produces docs GSD will consume.

LAYER 3 (Feature Execution)
GSD — /gsd-discuss → /gsd-plan → /gsd-execute → /gsd-ship
→ Run per feature. Consumes BMAD docs, executes in fresh contexts.

SYNERGY RESULT
✓ Enterprise planning depth (BMAD)
✓ Context-rot-free execution (GSD)
✓ TDD + review on every task (Superpowers)
